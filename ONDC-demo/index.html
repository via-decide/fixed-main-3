🧪 How This Demo Works
💳
UPI Payment Flow
The UPI ID and payment amount are real-looking but this is a UX prototype. No actual money is transferred. The UTR submission tests the form & duplicate-detection logic only.
Simulated — no real payment
🔢
UTR Validation
Enter any 12-digit number as a UTR. The app checks format (exactly 12 digits) and prevents duplicate submission within the same browser session via localStorage.
Stored in localStorage
🟢
Seller Confirmation Simulation
In production, the seller manually verifies UTR and updates the order status. Here, tap "Simulate Seller Confirm" to advance the order through the tracking timeline instantly.
Manual simulation button
📦
Order Tracking
Live tracking auto-refreshes every 10 seconds by checking localStorage for status changes. In production this would use real-time DB listeners.
localStorage polling
📄
PDF Receipt Download
After placing an order, you can download a PDF receipt from the tracking screen. Generated client-side using jsPDF — no server involved.
Client-side PDF (jsPDF)
☁️
Waitlist / Seller Interest / Feedback
These forms call Supabase Edge Functions if available. If the function fails (not deployed), data is saved locally and a "Saved locally (demo)" notice is shown.
Edge Function → localStorage fallback
🛒
E-commerce Products
All product listings, prices, stock counts, and discount percentages are hardcoded demo data. Rankings change based on your selected filters (Camera, Battery, etc.).
Hardcoded demo data
