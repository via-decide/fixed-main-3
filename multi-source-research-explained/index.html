Multi-Source Research: 4 Wikis, One Brief

ViaDecide Engineering 20 June 2025 5 min read

Most research tools give you a single source's perspective. Google gives you ten blue links. Wikipedia gives you one article. A language model gives you a confident paragraph that may or may not be grounded in anything real.

ViaDecide's research engine is built differently. One query fans out to four authoritative, openly licensed knowledge bases simultaneously — and returns a structured, source-attributed synthesis. Here's how.

Design principle: No hallucination. Every claim in a ViaDecide brief is attributed to a source the user can verify. Synthesis is clearly marked as synthesis. Contradiction between sources is surfaced, not hidden.

The four sources

📖
Wikipedia

Encyclopedic background, historical context, definitions, and general overview. The anchor source for most topics.

🗺️
Wikivoyage

Geographic and regional context. Essential for topics with a location dimension — agriculture, infrastructure, tourism, development.

📚
Wikibooks

Structured instructional content. Useful for how-to questions, process documentation, and technical procedures.

📰
Wikinews

Recent events and developments. Adds temporal context — what changed recently, current developments, breaking updates.

The query flow

When a user submits a research topic, the engine executes four parallel API calls — one to each Wikimedia endpoint — using the MediaWiki search API. Results are returned asynchronously and assembled into a unified response object.

// Simplified query fan-out const sources = [ { name: "Wikipedia", api: "en.wikipedia.org/w/api.php" }, { name: "Wikivoyage", api: "en.wikivoyage.org/w/api.php" }, { name: "Wikibooks", api: "en.wikibooks.org/w/api.php" }, { name: "Wikinews", api: "en.wikinews.org/w/api.php" } ]; const results = await Promise.allSettled( sources.map(s => fetchWikiArticle(s.api, topic)) );

Using Promise.allSettled (not Promise.all) is a deliberate choice. If one source is unavailable or returns no results for a topic, the other three still complete. The brief is built from whatever sources respond — with clear indication of which sources contributed.

Synthesis, not aggregation

The raw output from four sources is not a brief — it's four articles. The synthesis step extracts the key thesis from each source, identifies where sources agree, surfaces where they contradict or provide different angles, and assembles these into a structured brief with clear attribution.

Contradictions are not resolved — they're surfaced. A user researching water privatization policy will see that Wikipedia's encyclopedic overview and Wikinews's recent coverage may present different framings. That tension is valuable decision-relevant information, not noise to be averaged away.

Why open data matters

All four Wikimedia projects are licensed under Creative Commons — free to query, use, and build on. This keeps ViaDecide accessible at scale without licensing costs that would gate access for users in lower-income segments.

It also means the data is verifiable. Every cited source in a ViaDecide brief links back to a publicly accessible URL. A student can check our work. An official can verify a claim. A farmer can see where the information came from. Transparency is a feature, not an afterthought.
